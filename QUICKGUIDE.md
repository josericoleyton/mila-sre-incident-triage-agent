# QUICKGUIDE.md — Mila Quick Start

Get Mila running in 5 minutes.

---

## Step 1: Clone the Repository

```bash
git clone <repository-url>
cd mila
```

## Step 2: Configure Environment Variables

```bash
cp .env.example .env
```

Open `.env` and fill in your API keys:

| Variable | Where to get it |
|---|---|
| `LLM_MODEL` | Model string, e.g. `openrouter:google/gemma-4` or `claude-haiku-4-5-20251001` |
| `OPENROUTER_API_KEY` | [openrouter.ai](https://openrouter.ai) → API Keys (if using OpenRouter models) |
| `ANTHROPIC_API_KEY` | [console.anthropic.com](https://console.anthropic.com) → API Keys (if using Claude models) |
| `LINEAR_API_KEY` | Linear → Settings → API → Personal API Keys |
| `LINEAR_TEAM_ID` | Linear → Settings → Teams → copy Team ID |
| `LINEAR_WEBHOOK_SECRET` | Linear → Settings → API → Webhooks → create one pointing to your ngrok URL |
| `SLACK_BOT_TOKEN` | [api.slack.com](https://api.slack.com/apps) → OAuth & Permissions → Bot Token |
| `SLACK_CHANNEL_ID` | Slack → right-click channel → View channel details → copy Channel ID |
| `SLACK_WEBHOOK_URL` | Slack app → Incoming Webhooks → copy webhook URL |
| `GITHUB_TOKEN` | [github.com/settings/tokens](https://github.com/settings/tokens) → Generate new token (public repo read access) |
| `NGROK_AUTHTOKEN` | [dashboard.ngrok.com](https://dashboard.ngrok.com) → Your Authtoken |
| `LANGFUSE_SECRET_KEY` | Auto-generated by self-hosted Langfuse, or get from [cloud.langfuse.com](https://cloud.langfuse.com) |
| `LANGFUSE_PUBLIC_KEY` | Same as above |

> **Tip:** The defaults for `REDIS_URL` and Langfuse database settings work out of the box.

## Step 3: Build and Start All Services

```bash
docker compose up --build
```

Wait for all services to start. You will see health check confirmations for Redis and Langfuse DB.

## Step 4: Submit an Incident

Open **http://localhost:8080** in your browser.

Fill in the form:
- **Title:** `Orders stuck in submitted status for over 2 hours`
- **Description:** `Multiple orders are not progressing past the Submitted state. No errors visible in the dashboard. Customers may not be receiving their purchases.`
- **Component:** Order management
- **Severity:** High
- **Attachment:** Optionally upload a screenshot or log file

Click **Submit**. You will see a success screen with an incident ID.

## Step 5: Follow the Full Incident Flow

After submitting, Mila runs the complete pipeline automatically:

1. **Triage** — Mila analyzes the report, searches the eShop codebase, and classifies the incident
2. **Ticket created** — A new engineering ticket appears in your Linear workspace with root cause, affected files, severity, and suggested fix
3. **Team notified** — A Slack alert arrives in your configured channel with a summary and a link to the Linear ticket
4. **Reporter notified** — A Slack DM arrives confirming the incident has been escalated to the engineering team
5. **Resolution** — When the engineer marks the Linear ticket as **Done**, Mila detects the status change and sends a final Slack DM to the reporter confirming the incident is resolved

> **Note:** The ngrok tunnel must be running for Linear webhooks to reach the Ticket Service. Verify it is active at **http://localhost:4040**.

## Step 6: View Agent Traces

Open **http://localhost:3000** (Langfuse).

Sign in (first time: create an account on the local instance). Find the trace for your incident to see:
- The full prompt sent to the LLM
- Tool calls (code searches, file reads)
- Classification result with confidence and reasoning
- Token usage and latency

## Troubleshooting

| Problem | Solution |
|---|---|
| Services fail to start | Run `docker compose logs <service>` to check for missing env vars |
| No ticket created | Verify `LINEAR_API_KEY` and `LINEAR_TEAM_ID` in `.env` |
| No Slack messages | Verify `SLACK_BOT_TOKEN` and `SLACK_CHANNEL_ID`; ensure bot is invited to the channel |
| Resolution DM not received | Verify `NGROK_AUTHTOKEN` is set and the ngrok tunnel is running; check `LINEAR_WEBHOOK_SECRET` matches your Linear webhook config |
| Agent rate-limited | Switch to Claude: set `LLM_MODEL=claude-haiku-4-5-20251001` and add `ANTHROPIC_API_KEY` |
| Langfuse not loading | Wait 30s for Postgres to initialize; check `docker compose logs langfuse` |
